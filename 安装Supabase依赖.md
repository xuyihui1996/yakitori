当前项目已经包含了 Supabase 集成代码，但还**未安装 Supabase 依赖包**。

### 选择 1：继续使用 Mock 数据（当前状态）

如果你：
- 只是想本地测试功能
- 不需要多人协作
- 暂时不想接入真实后端

**什么都不用做**，当前代码会自动使用 Mock 数据。

✅ **已完成**：预设数据已清空，从干净状态开始  
✅ **当前状态**：可以正常运行，单人使用完全正常

### 选择 2：安装 Supabase 实现多人协作

如果你想要：
- 真正的多人点单功能
- 数据持久化（刷新不丢失）
- 跨设备同步

**请执行以下步骤**：

#### 1. 安装依赖

```bash
cd /home/kyo2/project/Ordered
npm install @supabase/supabase-js
```

#### 2. 配置 Supabase

按照 `接入Supabase指南.md` 的步骤：
1. 创建 Supabase 项目
2. 创建数据表
3. 配置环境变量
4. 切换到 Supabase 服务

#### 3. 重新构建

```bash
npm run build
```
